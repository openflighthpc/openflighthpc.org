--- heading: OpenFlight Azure CentOS 8 Image Import ---

Prerequisites

  • Azure CLI tool installed
  • Azure CLI tool logged into Azure account
  • A storage container within a resource group with a container for uploading the image to

Importing Image

  • Download image from the OpenFlight repo
  • Resize the system disk to a size suitable for applications and data (at least 16GB)
    • This requires v2+ of Qemu as it has support for Azure VHDs (tested with build of v3.1.0 on CentOS 7). The Qemu source is available here
    • Resize the image to desired size
      • qemu-img resize CENTOS8-OPENFLIGHT-2020.1-1803201919_azure.raw 16G
    • Convert disk to ensure Azure VHD support
      • qemu-img convert -p -f raw -O vpc -o subformat=fixed,force_size CENTOS8-OPENFLIGHT-2020.1-1803201919_azure.raw CENTOS8-OPENFLIGHT-2020.1-1803201919_azure-16G.raw
  • Upload resized image to a storage container
    • az storage blob upload --account-name mystorageaccount --container-name images --type page --file CENTOS8-OPENFLIGHT-2020.1-1803201919_azure-16G.raw --name CENTOS8-OPENFLIGHT-2020.1-1803201919_azure-16G.vhd
  • Create image from file
    • az image create --resource-group myresourcegroup --name CENTOS8-OPENFLIGHT-2020.1-1803201919_azure-16G --os-type Linux --source https://mystorageaccount.blob.core.windows.net/images/CENTOS8-OPENFLIGHT-2020.1-1803201919_azure-16G.vhd
  • The image is now available in the resource group to create VMs from
