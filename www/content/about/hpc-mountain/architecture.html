Architecture

This is the bottom of the HPC mountain. Architecture consists of the initial cluster design and choices related to what, where and how resources will be deployed.

Architecture design is the first and most critical step of cluster creation. It determines the platforms, tools and configuration of hardware and software. Key considerations of architecture are:

  • Deployment Platform - The resource solution for the cluster, whether it be cloud, on-premise metal or virtualisation.
  • Node Structure - The layout, purpose and differences between different node types in the cluster.
  • Network Configuration - The subnets, separate networks, VLANs and any other configuration details regarding system interconnect.
  • Operating System - The operating system to use and roughly how it will be considered (e.g. package groups to install).
  • Resistance and Redundancy - The required level of stability and systems in place to prevent loss of data.
  • Naming Conventions - The fully qualified domain name and logical structures applied to the nodes in the cluster to better identify them.
  • Security - The controls and restrictions in place to protect data and access to the system.
